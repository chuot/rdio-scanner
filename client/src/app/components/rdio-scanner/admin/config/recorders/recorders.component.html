Scanner Recorders are companion daemons (like tools/uniden-recorder) that feed audio from a physical scanner into rdio-scanner. Each recorder authenticates with its API key and pulls its runtime settings from this list. Hardware-local settings (serial port, audio device) live in the daemon's local .ini.

When disabled, the daemon will still authenticate but the server's disabled flag tells it not to write call files.

Friendly name shown in the admin UI. Pick something that identifies the physical scanner, e.g. "BCD996XT in basement".

Optional. If set, the daemon advertises this system to Auto-populate instead of relying on the WAV filename's #SYSLBL.

Output directory
Where the daemon writes WAVs. Must be readable by the DirWatch entry on this server (typically the same path). Leave blank to keep the daemon's local default.

Min silence (ms)
How long a carrier drop must last before the daemon closes the current call file. Increase if transmissions get split; decrease if back-to-back calls get glued.

Must be ≥ 0

Pre-roll (ms)
How much audio captured before squelch-open is prepended to each call so the first syllable isn't clipped.

Must be ≥ 0
